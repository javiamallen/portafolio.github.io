2. The Deliverable: A Prompt Engineering Case Study

The core asset was the *result* of a structured Prompt Engineering process. The goal was not just to ask for a presentation, but to *engineer* a specific, high-quality output from the AI.

Structured Prompt (YAML Configuration Style)
# Defines a structured data approach to prompting
---
role: "Senior Technical Advisor, Generative AI Specialist"
task: "Generate a 6-slide presentation outline."
topic: "Prompt Engineering for Business Value"

audience:
  type: "Technology Consultants"
  needs: "Business applications, not just academic theory."

constraints:
  - format: "6 slides"
  - content: "5 key points per slide"
  - length: "~6-8 words per point"
  - tone: "Professional, informative, direct"
---

This YAML-style approach demonstrates how I structure data for an AI, treating the model as a predictable API endpoint. It emphasizes **Role-Based Tasking**, **Audience Scoping**, and **Strict Output Constraints**.

3. The Stack: Engineering Skillset

Generating the asset was 20% of the task. The remaining 80% involved engineering the platform to demonstrate a robust technical skillset.

AI-Accelerated Development & Prompt Engineering

Leveraged Gemini (Google's AI) as a technical partner. Utilized **Prompt Engineering** for asset generation and Generative AI for code, debugging, and workflow acceleration.

Frontend Development (HTML, CSS & Bootstrap)

Structured the site with semantic HTML5, styled with custom CSS & Bootstrap 5, and implemented client-side validation using custom regex pattern for robust UX.

Dynamic Content & DOM Manipulation

Implemented **JavaScript DOM Manipulation** to dynamically inject randomized content (Wikipedia quotes) upon page load, demonstrating core frontend programming skills.

Responsive & Mobile-First Design

Implemented a "mobile-first" philosophy. Used Bootstrap's grid system, CSS Grid (minmax()), and the viewport meta tag to ensure the site is fully responsive and provides a flawless UX on all devices.

Version Control (Git)

Mastered the essential Git workflow: git clone, git add, git commit, and git push for robust source code management.

CI/CD & Deployment (GitHub Pages)

Took a local repository and successfully deployed it to a public production environment (javiamallen.github.io) using GitHub's CI/CD pipeline (Pages).

Backend Integration (BaaS)

Integrated a secure, functional contact form with Formspree, a Backend-as-a-Service (BaaS), to handle API-based data submission from a static site.

Project Management & Lifecycle

Managed the entire project lifecycle, from the initial (LinkedIn) challenge, through engineering (Git, Frontend, BaaS), to the final deployed product.
